═══════════════════════════════════════════════════════════════
    FLUJO DE COTIZACIÓN - PASO A PASO
═══════════════════════════════════════════════════════════════

1. CLIENTE CREA COTIZACIÓN
   • Agrega productos al carrito
   • Completa datos (nombre, email, WhatsApp)
   • Selecciona tipo de documento (Boleta/Factura)
   • Si es Factura: RUC, Razón Social, Dirección
   • Envía cotización
   → Estado: PENDING (Pendiente)
   → Se envía webhook a N8N (notificaciones automáticas)

2. COTIZADOR REVISA
   • Ve cotizaciones pendientes en panel
   • Puede ver detalles y PDF
   • DECISIÓN:
     ✓ APROBAR → Estado: APPROVED
       - Agrega días estimados de entrega (opcional)
       - Agrega notas (opcional)
     ✗ RECHAZAR → Estado: REJECTED
       - Debe indicar razón del rechazo

3. ADMIN AUTORIZA DESPACHO
   • Ve cotizaciones aprobadas
   • Al autorizar:
     - Verifica stock de productos
     - Si hay stock suficiente:
       • Descuenta stock automáticamente
       • Genera número de boleta/factura (ej: B-2024-000001)
       • Estado: AUTHORIZED
     - Si NO hay stock:
       • Muestra error, no autoriza

4. DESPACHO COMPLETADO
   • Admin marca como despachada
   → Estado: COMPLETED

═══════════════════════════════════════════════════════════════
    ESTADOS DE COTIZACIÓN
═══════════════════════════════════════════════════════════════

1. pending    → Pendiente de revisión
2. approved   → Aprobada por cotizador
3. rejected   → Rechazada por cotizador
4. authorized → Autorizada para despacho (stock descontado)
5. completed  → Despachada y completada
